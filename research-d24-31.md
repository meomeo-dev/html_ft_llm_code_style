已知上下文: 代码风格调优器，仅关心代码视觉外观，0-5量表

任务: 8个维度，每个给视觉外观重写 descriptions（0-5级，6个字符串）：

测试类:
A: 测试覆盖率 - 0:无测试 / 1:核心模块少量测试 / 2:50%覆盖率 / 3:80%覆盖率 / 4:95%分支覆盖 / 5:100%含分支条件
B: 测试独立性 - 0:测试可依赖共享状态 / 1:不共享可变状态但可依赖数据库 / 2:每个测试准备自身数据 / 3:每个测试完全独立 / 4:测试严格隔离用内存数据库 / 5:完全隔离沙盒环境
C: 断言风格 - 0:不使用断言仅用打印 / 1:简单assert语句 / 2:框架默认断言 / 3:BDD风格expect / 4:链式匹配器 / 5:声明式断言库生成行为说明书
D: 模拟使用 - 0:不使用模拟直接调用真实实现 / 1:仅外部服务用桩 / 2:数据库网络用模拟 / 3:所有单元测试外部依赖模拟 / 4:依赖注入与模拟框架精细控制 / 5:严格模拟一切外部边界

性能类:
E: 算法效率 - 0:不关心复杂度 / 1:对O(n^2)以上优化 / 2:确保主要流程线性或对数级别 / 3:选择已知最优算法 / 4:主动分析复杂度并记录 / 5:严格证明算法复杂度
F: 过早优化抑制 - 0:随时优化 / 1:尽量保持高效 / 2:先清晰再优化 / 3:只优化剖析过的热点 / 4:禁止无测量数据的优化 / 5:严格先正确再清晰后优化
G: 资源管理 - 0:手动开关资源 / 1:try-finally确保关闭 / 2:with语句或using块 / 3:RAII或Disposable模式 / 4:所有资源类实现AutoCloseable / 5:容器或智能指针完全自动管理
H: 内存管理 - 0:随意分配不关心回收 / 1:避免明显大对象长期持有 / 2:注意集合初始化大小 / 3:主动使用弱引用缓存策略 / 4:使用内存分析工具监控 / 5:严格防范泄漏与过度分配

用视觉语言描述屏幕上的外观，不描述规则本身。
